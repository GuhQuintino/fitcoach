# Product Requirements Document (PRD) - Fitcoach Pro

## 1. Product Overview
Fitcoach Pro is a premium fitness coaching platform connecting coaches and students. It manages workouts, assessments, subscription plans, and feedback with a focus on a refined visual experience and high performance. The goal is to provide a "native app" feel via PWA capabilities and a glassmorphism-based design.

## 2. User Roles & Personas
*   **Admin**: Platform manager. Responsible for approving new Coaches.
*   **Coach**: Fitness professional. manages students, creates workout plans, creates exercises, and monitors student progress.
*   **Student**: End-user. Receives workout plans, logs performance, tracks weight and photos, and communicates with the coach.

## 3. Core Features

### 3.1. Authentication & Onboarding
*   **Login**: Email/Password authentication via Supabase.
*   **Registration (Coach)**: Open registration, but starts in `pending` state. Requires Admin approval. Must capture CREF ID (format: `000000-G/UF`).
*   **Registration (Student)**: Invitation-only. Must register via a unique link generated by a Coach (`/register?coach=UUID`). Starts as `pending`, requires Coach approval.
*   **Access Control**: Role-based access control (RBAC) protecting all routes.

### 3.2. Coach Features
*   **Dashboard**: Overview of active students and subscription status.
*   **Student Management**: View list of students, approve pending students, view student profiles.
*   **Workout Management**: Create and edit workout routines (`/editor`), manage personal exercise library (`/library`).
*   **Invite System**: Generate invitation links (or QR codes) for new students.
*   **Plans**: View own subscription status.

### 3.3. Student Features
*   **Dashboard**: View daily workout, weekly progress summary.
*   **Workout Execution**: Interactive player for executing workouts. Features: video player, stopwatch, logging of weights/reps (`/workout/:id`).
*   **History**: View logs of past workouts.
*   **Profile**: Manage body measurements, upload progress photos (Evolution Gallery).

### 3.4. Admin Features
*   **Dashboard**: Approve or reject new Coach sign-ups.

### 3.5. Subscription & Access Rules
*   **Expiration**: If a subscription (Coach or Student) expires, access is revoked. Redirects to `/subscription-expired`.
*   **Cascading Lock**: If a Coach's subscription expires, ALL their students lose access immediately.
*   **Manual Renewal**: No in-app payment. Renewal involves contacting support via WhatsApp button.

## 4. User Experience & Design
*   **Aesthetics**: Glassmorphism, tailored gradients (sky/indigo/emerald), ample whitespace, rounded corners.
*   **Interactivity**: Micro-animations on hover/click.
*   **PWA**: Installable on mobile, offline-ready (manifest configured), standalone execution.

## 5. Technical Requirements

### 5.1. Frontend
*   **Stack**: React 19, Vite 6, TypeScript.
*   **Styling**: Tailwind CSS v4.
*   **State**: React Context API (`AuthContext`).
*   **Routing**: React Router Dom v7 (HashRouter).

### 5.2. Backend (Supabase)
*   **Auth**: Supabase Auth (Email/Password).
*   **Database**: PostgreSQL.
*   **Storage**: Storing images (evolution photos).
*   **Security**: Row Level Security (RLS) is mandatory for data isolation.

### 5.3. Schema Overview
*   `profiles`: Base user data (id, role, email).
*   `coaches_data`: Coach usage details (cref, is_approved).
*   `students_data`: Student specific details (objectives, coach_id).
*   `exercises`: Database of exercises.
*   `routines`: Workout templates.
*   `workout_logs` & `set_logs`: Execution history.

## 6. Integrations
*   **YouTube**: Automated script (`scripts/youtube-upload`) uploads generic exercise videos to YouTube as "Unlisted" and syncs ID to database.
*   **WhatsApp**: Standardized links for support and coach-student communication (must use `+55` prefix).
