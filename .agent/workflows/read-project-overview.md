description: Ler a documentação do projeto antes de fazer qualquer nova implementação
---

Sempre que o usuário solicitar uma nova funcionalidade, alteração de regra de negócio ou refatoração, siga estes passos:

1. // turbo
   Leia o arquivo `PROJECT_OVERVIEW.md` no root do projeto para entender o contexto atual, a pilha tecnológica e as regras de negócio.
2. Analise se a nova solicitação impacta alguma das camadas documentadas (Frontend, Backend, Database).
3. Se houver mudanças na arquitetura, atualize o arquivo `PROJECT_OVERVIEW.md` após concluir a tarefa.
